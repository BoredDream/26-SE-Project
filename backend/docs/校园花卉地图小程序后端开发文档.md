# 校园花卉地图小程序后端开发文档

## 1. 项目概述

### 1.1 项目背景
本项目旨在为校园师生提供一个专属的「校园花卉实时地图」小程序，整合地点信息、花期状态与社区分享。通过用户打卡众包机制保证花期信息的时效性，形成“地图查询 - 花期提醒 - 社区分享”的一站式体验。

### 1.2 核心目标
- **实时准确**：利用用户打卡众包机制，确保花期信息实时更新
- **数据自驱**：用户行为即数据，打卡即内容，形成自驱动飞轮
- **低运营成本**：自动化状态更新与内容审核，减少人工干预

## 3. 数据库设计

### 3.1 核心表结构

#### 3.1.1 用户表 (`users`)
```sql
CREATE TABLE users (
  id          INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  openid      VARCHAR(64)  NOT NULL UNIQUE COMMENT '微信openid',
  nickname    VARCHAR(50)  NOT NULL DEFAULT '',
  avatar_url  VARCHAR(500) DEFAULT NULL,
  role        ENUM('user','admin') NOT NULL DEFAULT 'user',
  created_at  DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at  DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

#### 3.1.2 花卉地点表 (`locations`)
```sql
CREATE TABLE locations (
  id                     INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  name                   VARCHAR(100)  NOT NULL COMMENT '地点名称',
  description            TEXT          DEFAULT NULL,
  latitude               DECIMAL(10,7) NOT NULL,
  longitude              DECIMAL(10,7) NOT NULL,
  flower_species         VARCHAR(100)  NOT NULL COMMENT '花卉品种',
  bloom_status           ENUM('dormant','budding','blooming','withering') NOT NULL DEFAULT 'dormant',
  historical_bloom_start VARCHAR(20)   DEFAULT NULL COMMENT '如 03-01',
  historical_bloom_end   VARCHAR(20)   DEFAULT NULL COMMENT '如 03-20',
  cover_image            VARCHAR(500)  DEFAULT NULL,
  checkin_count          INT UNSIGNED  NOT NULL DEFAULT 0 COMMENT '用于热力图',
  status_updated_at      DATETIME      DEFAULT NULL,
  created_at             DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

#### 3.1.3 用户打卡表 (`checkins`)
```sql
CREATE TABLE checkins (
  id             INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  user_id        INT UNSIGNED NOT NULL,
  location_id    INT UNSIGNED NOT NULL,
  bloom_report   ENUM('budding','blooming','withering') DEFAULT NULL COMMENT '用户报告的花期状态',
  content        TEXT         DEFAULT NULL COMMENT '图文描述，最多500字',
  images         JSON         DEFAULT NULL COMMENT '三级图片URL对象数组',
  likes_count    INT UNSIGNED NOT NULL DEFAULT 0,
  gps_verified   TINYINT(1)   NOT NULL DEFAULT 0 COMMENT 'GPS校验是否通过',
  user_latitude  DECIMAL(10,7) DEFAULT NULL,
  user_longitude DECIMAL(10,7) DEFAULT NULL,
  audit_status   ENUM('pending','approved','rejected') NOT NULL DEFAULT 'pending',
  is_visible     TINYINT(1)    NOT NULL DEFAULT 1,
  created_at     DATETIME      NOT NULL DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id)     REFERENCES users(id),
  FOREIGN KEY (location_id) REFERENCES locations(id),
  INDEX idx_location_created (location_id, created_at DESC),
  INDEX idx_audit_status (audit_status)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

#### 3.1.4 订阅表 (`subscriptions`)

CREATE TABLE subscriptions (

### 3.2 ER关系说明
- **users → checkins**: 1:N，一个用户可发布多条打卡
- **locations → checkins**: 1:N，一个地点可有多条打卡
- **users → subscriptions**: 1:N，一个用户可订阅多个地点
- **users → achievements**: 1:N，一个用户可获得多个花种成就（每种花唯一）
- **locations → pioneer_honors**: 1:N，一个地点每次花期只有一个探花使

## 4. API接口设计

### 4.1 接口规范
- **Base URL**: `https://api.your-domain.com/v1`
- **请求格式**: `Content-Type: application/json`
- **鉴权方式**: JWT Bearer Token，置于 `Authorization` Header
- **统一响应格式**:
```json
{
  "code": 0,
  "message": "ok",
  "data": { ... }
}
```

### 4.2 错误码定义
| code | 含义 |
|------|------|
| 0 | 成功 |
| 1001 | 参数错误 |
| 1002 | 未登录 / Token过期 |
| 1003 | 无权限 |
| 1004 | 资源不存在 |
| 5000 | 服务器内部错误 |

### 4.3 核心接口列表

#### 4.3.1 用户模块
- **POST /auth/login** - 微信登录
  - 请求: `{ code: string }` (wx.login()返回的临时code)
  - 响应: `{ token: string, user: { id, nickname, role } }`

#### 4.3.2 地点模块
- **GET /locations** - 获取地点列表
  - Query: `status`(可选，花期筛选), `keyword`(可选，搜索)
  - 响应: 地点数组，含花期状态、打卡数等
  
- **GET /locations/:id** - 获取地点详情
  - 返回地点完整信息 + 最近10条已审核通过的打卡记录
  
- **PATCH /locations/:id/status** - 更新花期状态（仅管理员）
  - 请求: `{ bloom_status: 'dormant' | 'budding' | 'blooming' | 'withering' }`

#### 4.3.3 打卡模块
- **POST /checkins** - 发布打卡（需登录）
  - 请求参数:
    - `location_id` (integer, 必填): 地点ID
    - `bloom_report` (string, 选填): 花期状态
    - `images` (array, 必填): 图片对象数组，最多9组
    - `content` (string, 选填): 图文描述，最多500字
    - `user_latitude/longitude` (number, 选填): GPS坐标
  - 后置处理: 异步触发AI审核、GPS校验、花期投票、成就检查

- **GET /locations/:id/checkins** - 获取地点打卡列表
  - Query: `page`, `pageSize`
  - 仅返回 `audit_status=approved` 的记录

- **POST /checkins/:id/like** - 点赞（需登录）
  - 幂等操作，重复点赞不报错

- **POST /checkins/:id/report** - 举报打卡（需登录）
  - 请求: `{ reason: string }`

#### 4.3.4 订阅模块
- **POST /subscriptions** - 订阅地点（需登录）
- **DELETE /subscriptions/:locationId** - 取消订阅
- **GET /subscriptions** - 我的订阅列表（需登录）

#### 4.3.5 上传模块
- **POST /upload/image** - 上传图片（需登录）
  - Content-Type: `multipart/form-data`
  - 单文件 ≤ 10MB，格式限 jpg/png/webp
  - 响应: 三级分辨率图片URL对象
```json
{
  "original": "https://cos.../original/xxx.webp",
  "standard": "https://cos.../standard/xxx.jpg",
  "thumb": "https://cos.../thumb/xxx.jpg"
}
```

#### 4.3.6 成就与称号模块
- **GET /users/me/achievements** - 我的花圃（需登录）
- **GET /users/me/titles** - 我的称号（需登录）
- **GET /users/:id/achievements** - 查看他人花圃（公开）

#### 4.3.7 排行榜模块
- **GET /leaderboard/checkins** - 打卡王榜
- **GET /leaderboard/likes** - 点赞榜
- **GET /leaderboard/heatmap** - 地图热力数据

#### 4.3.8 管理模块（仅管理员）
- **GET /admin/checkins/pending** - 待复核举报列表
- **PATCH /admin/checkins/:id/audit** - 审核操作
  - 请求: `{ action: 'approve' | 'reject', reason?: string }`

## 5. 核心业务逻辑

### 5.1 GPS校验机制
使用Haversine公式计算用户打卡位置与目标地点的距离：
```javascript
function haversineDistance(lat1, lon1, lat2, lon2) {
  const R = 6371000; // 地球半径(米)
  const dLat = (lat2 - lat1) * Math.PI / 180;
  const dLon = (lon2 - lon1) * Math.PI / 180;
  const a = Math.sin(dLat/2) ** 2 +
            Math.cos(lat1 * Math.PI/180) * Math.cos(lat2 * Math.PI/180) *
            Math.sin(dLon/2) ** 2;
  return R * 2 * Math.atan2(Math.sqrt(a), Math.sqrt(1 - a));
}

const GPS_RADIUS = 150; // 米
const gpsVerified = haversineDistance(userLat, userLon, locLat, locLon) <= GPS_RADIUS;
```
- **gps_verified = true**: 可获得金色成就
- **gps_verified = false**: 仅获得银色成就

### 5.2 AI内容审核（异步）
打卡写入数据库后异步执行，不阻塞用户发布：
```javascript
async function auditCheckin(checkinId, images, content) {
  try {
    // 图片审核
    for (const img of images) {
      const r = await wxCloud.imgSecCheck({ imgUrl: img.standard });
      if (r.errCode !== 0) {
        await setAuditStatus(checkinId, 'rejected', '图片包含违规内容');
        return;
      }
    }
    // 文字审核
    if (content) {
      const r = await wxCloud.msgSecCheck({ content });
      if (r.errCode !== 0) {
        await setAuditStatus(checkinId, 'rejected', '文字包含违规内容');
        return;
      }
    }
    // 审核通过
    await setAuditStatus(checkinId, 'approved', null);
    await triggerPostCheckinLogic(checkinId);
  } catch (e) {
    // 审核服务异常时默认通过，避免误伤
    await setAuditStatus(checkinId, 'approved', null);
  }
}
```

### 5.3 花期众包投票机制
审核通过后执行，基于24小时内的用户报告自动更新花期状态：
```javascript
async function updateBloomStatusByVote(locationId) {
  const since = new Date(Date.now() - 24 * 60 * 60 * 1000);
  const reports = await db.query(
    `SELECT bloom_report, COUNT(*) AS cnt FROM checkins
     WHERE location_id=? AND bloom_report IS NOT NULL
     AND audit_status='approved' AND created_at>?
     GROUP BY bloom_report`,
    [locationId, since]
  );
  
  const total = reports.reduce((s, r) => s + r.cnt, 0);
  for (const r of reports) {
    // 条件：至少3条报告 且 占比≥60%
    if (r.cnt >= 3 && r.cnt / total >= 0.6) {
      await db.query(
        'UPDATE locations SET bloom_status=?, status_updated_at=NOW() WHERE id=?',
        [r.bloom_report, locationId]
      );
      // 若变为盛开，通知订阅者
      if (r.bloom_report === 'blooming') {
        await notifySubscribers(locationId);
      }
      break;
    }
  }
}
```

### 5.4 成就触发流程
审核通过后异步执行：
1. 查询用户是否已有该花种成就
2. 若无 → 插入新记录，`grade`由`gps_verified`决定（true→gold / false→silver）
3. 若已有silver且本次`gps_verified=true` → 升级为gold
4. 检查探花使：若该地点本花期无记录，则插入`pioneer_honors`
5. 检查称号解锁条件

### 5.5 称号自动解锁规则
| title_key | 称号名 | 触发条件 |
|-----------|--------|----------|
| `first_checkin` | 🌱 花卉初探者 | 用户打卡总数 = 1 |
| `checkin_10` | 🌸 花卉观察员 | 用户打卡总数 = 10 |
| `likes_50` | 📸 最美摄影师 | 单条打卡获赞 ≥ 50 |
| `pioneer` | 🔍 探花使 | 成为某地点当季首个打卡者 |
| `four_seasons` | 🌳 四季守望者 | 同一地点春夏秋冬各≥1次打卡 |
| `king_monthly` | 👑 打卡王 | 每月打卡数最高者（定时任务） |

**防重机制**: `titles`表的`UNIQUE KEY (user_id, title_key)`保证同一称号不重复授予。

### 5.6 图片分级存储
使用`sharp`库生成三级分辨率并上传至COS：
```javascript
async function processAndUpload(buffer, filename) {
  const [original, standard, thumb] = await Promise.all([
    sharp(buffer).webp({ quality: 92 }).toBuffer(),
    sharp(buffer).resize({ width: 1080, height: 1080, fit: 'inside' }).jpeg({ quality: 85 }).toBuffer(),
    sharp(buffer).resize({ width: 400, height: 400, fit: 'inside' }).jpeg({ quality: 75 }).toBuffer(),
  ]);
  
  const [oriUrl, stdUrl, thumbUrl] = await Promise.all([
    uploadToCOS(`original/${filename}.webp`, original),
    uploadToCOS(`standard/${filename}.jpg`, standard),
    uploadToCOS(`thumb/${filename}.jpg`, thumb),
  ]);
  
  return { original: oriUrl, standard: stdUrl, thumb: thumbUrl };
}
```
图片对象以JSON格式存储于`checkins.images`字段。

### 5.7 微信订阅消息推送
| 场景 | 模板字段 | 防重逻辑 |
|------|----------|----------|
| 花期盛开通知 | 地点名/花种/状态文案/更新时间 | Redis key: `notify:{locationId}:{year}`，同一花期只推一次 |

## 6. 安全规范

### 6.1 内容安全
- **图片审核**: 调用微信云`imgSecCheck`接口，不通过则拒绝存储
- **文字审核**: 调用`msgSecCheck`接口，不通过则拒绝发布
- **异常处理**: 审核服务异常时默认通过，避免误伤，但记录日志
- **人工干预**: 管理员后台可下架内容、封禁用户

### 6.2 接口安全
- **鉴权**: 所有写操作必须携带有效JWT Token
- **权限控制**: 管理员接口额外校验`role === 'admin'`
- **频率限制**: 
  - 图片上传：单文件≤10MB，每用户每小时最多20张
  - 所有接口启用`express-rate-limit`防止滥用

### 6.3 数据安全
- **隐私保护**: 用户`openid`不对外暴露，API响应仅返回`user_id`
- **Token管理**: JWT有效期7天，敏感操作需重新鉴权
- **密钥管理**: 所有密钥通过环境变量注入，`.env`文件禁止提交至代码仓库

## 7. 部署方案

### 7.1 推荐方案：微信云开发
- **云函数**: Node.js业务逻辑部署为云函数，自动扩缩容
- **云数据库**: 托管MySQL，无需自行运维
- **云存储**: 替代COS，用于图片存储
- **优势**: 与微信生态深度集成，免运维，适合校园级小程序快速上线

### 7.2 备选方案：自建服务器
| 步骤 | 操作 |
|------|------|
| 1. 服务器 | 购买云服务器（2核4G），部署Ubuntu 22.04 |
| 2. 环境 | 安装Node.js 18、MySQL 8.0、Redis 7.0、Nginx |
| 3. 进程管理 | `pm2 start app.js --name flower-api` |
| 4. 反向代理 | Nginx配置HTTPS，将`/v1`代理到Node服务 |
| 5. SSL证书 | Let's Encrypt免费证书（certbot） |
| 6. 域名配置 | 微信公众平台配置服务器域名白名单 |

### 7.3 小程序发布流程
1. 微信开发者工具 → 上传代码（填写版本号和描述）
2. 微信公众平台 → 版本管理 → 提交审核
3. 审核通过（通常1-3个工作日）→ 发布上线
4. 通知用户搜索小程序名称或扫码体验

| 未登录打卡 | 不携带Token调用`POST /checkins` | 返回1002未登录错误 |
| 花期填写可选 | `bloom_report`字段留空发布打卡 | 发布成功，不参与花期投票统计 |
| 称号去重 | 用户第10次打卡后重复触发检查 | 称号表仅有一条`checkin_10`记录 |
| 成就升级 | 已有silver成就的花种，新增GPS验证打卡 | 成就`grade`升级为gold |
