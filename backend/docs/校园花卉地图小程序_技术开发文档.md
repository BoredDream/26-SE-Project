### 2.1 完整表结构

#### 用户表 `users`

```sql
CREATE TABLE users (
  id          INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  openid      VARCHAR(64)  NOT NULL UNIQUE COMMENT '微信 openid',
  nickname    VARCHAR(50)  NOT NULL DEFAULT '',
  avatar_url  VARCHAR(500) DEFAULT NULL,
  role        ENUM('user','admin') NOT NULL DEFAULT 'user',
  created_at  DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at  DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

#### 花卉地点表 `locations`

```sql
CREATE TABLE locations (
  id                     INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  name                   VARCHAR(100)  NOT NULL COMMENT '地点名称',
  description            TEXT          DEFAULT NULL,
  latitude               DECIMAL(10,7) NOT NULL,
  longitude              DECIMAL(10,7) NOT NULL,
  flower_species         VARCHAR(100)  NOT NULL COMMENT '花卉品种',
  flower_species_en      VARCHAR(100)  DEFAULT NULL,
  bloom_status           ENUM('dormant','budding','blooming','withering') NOT NULL DEFAULT 'dormant',
  historical_bloom_start VARCHAR(20)   DEFAULT NULL COMMENT '如 03-01',
  historical_bloom_end   VARCHAR(20)   DEFAULT NULL COMMENT '如 03-20',
  cover_image            VARCHAR(500)  DEFAULT NULL,

  status_updated_at      DATETIME      DEFAULT NULL,
  created_at             DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

#### 用户打卡表 `checkins`

```sql
CREATE TABLE checkins (
  id             INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  user_id        INT UNSIGNED NOT NULL,
  location_id    INT UNSIGNED NOT NULL,
  bloom_report   ENUM('budding','blooming','withering') DEFAULT NULL COMMENT '用户填写的花期状态，可为空',
  content        TEXT         DEFAULT NULL COMMENT '图文描述，最多500字',
  images         JSON         DEFAULT NULL COMMENT '三级图片URL对象数组',
  likes_count    INT UNSIGNED NOT NULL DEFAULT 0,
  gps_verified   TINYINT(1)   NOT NULL DEFAULT 0 COMMENT 'GPS校验是否通过',
  user_latitude  DECIMAL(10,7) DEFAULT NULL,
  user_longitude DECIMAL(10,7) DEFAULT NULL,
  audit_status   ENUM('pending','approved','rejected') NOT NULL DEFAULT 'pending',
  audit_reason   VARCHAR(200)  DEFAULT NULL,
  is_visible     TINYINT(1)    NOT NULL DEFAULT 1,
  created_at     DATETIME      NOT NULL DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id)     REFERENCES users(id),
  FOREIGN KEY (location_id) REFERENCES locations(id),
```

## 3. 后端 API 接口文档

### 3.1 接口规范

1. **Base URL**：`https://api.your-domain.com/v1`
2. **请求格式**：`Content-Type: application/json`
3. **鉴权**：JWT Bearer Token，登录后获取，置于 `Authorization` Header
4. **统一响应格式**：

```json
{
  "code": 0,         // 0=成功，非0=失败
  "message": "ok",
  "data": { ... }
}
```

#### 错误码

| code | 含义             |
| :--- | :------------- |
| 0    | 成功             |
| 1001 | 参数错误           |
| 1002 | 未登录 / Token 过期 |
| 1003 | 无权限            |
| 1004 | 资源不存在          |
| 5000 | 服务器内部错误        |

### 3.2 用户模块

- **POST /auth/login** — 微信登录
  - 请求：`{ code: string }`（wx.login() 返回的临时 code）
  - 响应：`{ token: string, user: { id, nickname, role } }`

### 3.3 地点模块

- **GET /locations** — 获取地点列表
  - Query：`status`（可选，花期筛选）、`keyword`（可选，搜索）
  - 响应：地点数组，含 id / name / latitude / longitude / flower\_species / bloom\_status / cover\_image / checkin\_count / status\_updated\_at
- **GET /locations/:id** — 获取地点详情
  - 返回地点完整信息 + 最近10条已审核通过的打卡记录（含图片）。
- **PATCH /locations/:id/status** — 更新花期状态（仅管理员）
  - 请求：`{ bloom_status: 'dormant' | 'budding' | 'blooming' | 'withering' }`

### 3.4 打卡模块

- **POST /checkins** — 发布打卡（需登录）

| 字段              | 类型      | 必填 | 说明                                      |
| :-------------- | :------ | :- | :-------------------------------------- |
| location\_id    | integer | 是  | 地点 ID                                   |
| bloom\_report   | string  | 否  | 花期状态：budding / blooming / withering，可不填 |
| images          | array   | 是  | 图片对象数组（先调用上传接口），最多9组                    |
| content         | string  | 否  | 图文描述，最多500字                             |
| user\_latitude  | number  | 否  | 用户打卡时 GPS 纬度                            |
| user\_longitude | number  | 否  | 用户打卡时 GPS 经度                            |

> 发布后异步触发：① AI 内容审核 ② GPS 校验 ③ 花期投票 ④ 成就检查

- **GET /locations/:id/checkins** — 获取地点打卡列表
  - Query：`page`（默认1）、`pageSize`（默认10，最大20）。仅返回 `audit_status=approved` 的记录，按 `created_at` 倒序。
- **POST /checkins/:id/like** — 点赞（需登录）
  - 幂等操作，重复点赞不报错。点赞后触发「最美摄影师」称号检查。
- **POST /checkins/:id/report** — 举报打卡（需登录）
  - 请求：`{ reason: string }`。每用户每条内容只能举报一次，举报后进入人工复核队列。

### 3.5 订阅模块

- **POST /subscriptions** — 订阅地点（需登录）
  - 请求：`{ location_id: integer }`。幂等操作。
- **DELETE /subscriptions/:locationId** — 取消订阅（需登录）
- **GET /subscriptions** — 我的订阅列表（需登录）
  - 返回订阅的所有地点信息，含当前花期状态。

### 3.6 上传模块

- **POST /upload/image** — 上传图片（需登录）

```json

    "original": "https://cos.../original/xxx.webp",
    "standard": "https://cos.../standard/xxx.jpg",
    "thumb":    "https://cos.../thumb/xxx.jpg"
  }

```

- **GET /users/me/achievements** — 我的花圃（需登录）

```json
{
  "code": 0,
```

- **GET /users/me/titles** — 我的称号（需登录）

- **GET /users/:id/achievements** — 查看他人花圃（公开）

### 3.8 排行榜模块

- **GET /leaderboard/checkins** — 打卡王榜
  - Query：`month`（格式 2026-03，默认当月）。返回前20名，含 rank / user\_id / nickname / avatar\_url / checkin\_count。
- **GET /leaderboard/likes** — 点赞榜

### 3.9 内容审核模块（仅管理员）

- **GET /admin/checkins/pending** — 待复核举报列表
  - 返回被举报、待人工复核的打卡内容列表。
- **PATCH /admin/checkins/:id/audit** — 审核操作
  - 请求：`{ action: 'approve' | 'reject', reason?: string }`

## 4. 核心业务逻辑

### 4.1 GPS 校验

打卡接口接收用户 GPS 坐标后，使用 Haversine 公式计算与目标地点的距离：

```javascript
// utils/geo.js
function haversineDistance(lat1, lon1, lat2, lon2) {
  const R = 6371000;
  const dLat = (lat2 - lat1) * Math.PI / 180;
  const dLon = (lon2 - lon1) * Math.PI / 180;
  const a = Math.sin(dLat/2) ** 2 +
    Math.cos(lat1 * Math.PI/180) * Math.cos(lat2 * Math.PI/180) *
    Math.sin(dLon/2) ** 2;
  return R * 2 * Math.atan2(Math.sqrt(a), Math.sqrt(1 - a));
}

const GPS_RADIUS = 150; // 米
const gpsVerified = haversineDistance(
  userLat, userLon, location.latitude, location.longitude
) <= GPS_RADIUS;
```

### 4.2 AI 内容审核（异步）

打卡写入数据库后异步执行，不阻塞用户发布操作：

```javascript
async function auditCheckin(checkinId, images, content) {
  try {
    for (const img of images) {
      const r = await wxCloud.imgSecCheck({ imgUrl: img.standard });
      if (r.errCode !== 0) {
        await setAuditStatus(checkinId, 'rejected', '图片包含违规内容');
        return;
      }
    }
    if (content) {
      const r = await wxCloud.msgSecCheck({ content });
      if (r.errCode !== 0) {
        await setAuditStatus(checkinId, 'rejected', '文字包含违规内容');
        return;
      }
    }
    await setAuditStatus(checkinId, 'approved', null);
    await triggerPostCheckinLogic(checkinId);
  } catch (e) {
    await setAuditStatus(checkinId, 'approved', null); // 审核异常默认通过
  }
}
```

### 4.3 花期众包投票

审核通过后执行，仅统计填写了 `bloom_report` 的打卡：

```javascript
async function updateBloomStatusByVote(locationId) {
  const since = new Date(Date.now() - 24 * 60 * 60 * 1000);
  const reports = await db.query(
    `SELECT bloom_report, COUNT(*) AS cnt FROM checkins
     WHERE location_id=? AND bloom_report IS NOT NULL
     AND audit_status='approved' AND created_at>?
     GROUP BY bloom_report`,
    [locationId, since]
  );
  const total = reports.reduce((s, r) => s + r.cnt, 0);
  for (const r of reports) {
    if (r.cnt >= 3 && r.cnt / total >= 0.6) {
      await db.query(
        'UPDATE locations SET bloom_status=?, status_updated_at=NOW() WHERE id=?',
        [r.bloom_report, locationId]
      );
      if (r.bloom_report === 'blooming') await notifySubscribers(locationId);
      break;
    }
  }
}
```

### 4.4 成就触发流程

审核通过后异步执行：

1. 查询该用户是否已有该花种成就记录
2. 若无 → 插入新记录，grade 由 `gps_verified` 决定（true→gold / false→silver）
3. 若已有 silver 且本次 `gps_verified=true` → UPDATE grade 为 gold
4. 若已有 gold → 仅更新该花种打卡计数，不改变成就
5. 检查探花使：查询 `pioneer_honors` 是否已有本地点本花期记录，若无则插入
6. 检查称号解锁（见 4.5）

### 4.5 称号自动解锁规则

| title\_key     | 称号名      | 触发时机与条件                                 |
| :------------- | :------- | :-------------------------------------- |
| first\_checkin | 🌱 花卉初探者 | 每次打卡后检查：该用户 checkins 总数 = 1             |
| checkin\_10    | 🌸 花卉观察员 | 每次打卡后检查：该用户 checkins 总数 = 10            |
| likes\_50      | 📸 最美摄影师 | 每次点赞后检查：被点赞的 checkin.likes\_count >= 50 |
| pioneer        | 🔍 探花使   | pioneer\_honors 表插入新记录时触发               |
| four\_seasons  | 🌳 四季守望者 | 每次打卡后检查：同一地点是否有春夏秋冬各≥1次打卡               |
| king\_monthly  | 👑 打卡王   | 每月1日定时任务统计上月打卡数最高者授予                    |

> 称号解锁后推送微信订阅消息通知用户。同一称号不重复授予（UNIQUE KEY 约束保证）。

### 4.6 图片分级存储

```javascript
const sharp = require('sharp');
async function processAndUpload(buffer, filename) {
  const [original, standard, thumb] = await Promise.all([
    sharp(buffer).webp({ quality: 92 }).toBuffer(),
    sharp(buffer).resize({ width: 1080, height: 1080, fit: 'inside', withoutEnlargement: true }).jpeg({ quality: 85 }).toBuffer(),
    sharp(buffer).resize({ width: 400, height: 400, fit: 'inside', withoutEnlargement: true }).jpeg({ quality: 75 }).toBuffer(),
  ]);
  const [oriUrl, stdUrl, thumbUrl] = await Promise.all([
    uploadToCOS(`original/${filename}.webp`, original),
    uploadToCOS(`standard/${filename}.jpg`, standard),
    uploadToCOS(`thumb/${filename}.jpg`, thumb),
  ]);
  return { original: oriUrl, standard: stdUrl, thumb: thumbUrl };
}
```

`[ { "original": "...", "standard": "...", "thumb": "..." }, ... ]`

### 5.4 网络请求封装

```javascript
// utils/request.js
const BASE_URL = 'https://api.your-domain.com/v1';
export function request(method, path, data = {}) {
  return new Promise((resolve, reject) => {
    wx.request({
      url: BASE_URL + path, method, data,
      header: {
        'Content-Type': 'application/json',
        'Authorization': `Bearer ${wx.getStorageSync('token')}`
      },
      success: (res) => {
        if (res.data.code === 0)    resolve(res.data.data);
        else if (res.data.code === 1002) handleTokenExpired();
        else reject(res.data);
      },
      fail: reject
    });
  });
}
```

### 5.5 打卡页关键逻辑

1. 花期状态选择为可选项，入口文案：「当前花期（选填，帮助大家了解花期）」
2. 页面加载时静默调用 `wx.getLocation`，用于 GPS 校验，不向用户展示结果
3. 用户拒绝定位时继续允许发布，`user_latitude` / `user_longitude` 传 null
4. 发布成功后，若响应中含 `new_achievement` 字段则展示「获得新成就」弹窗

```javascript
// 静默获取 GPS（utils/geo.js）
export function getLocationSilently() {
  return new Promise((resolve) => {
    wx.getLocation({
      type: 'gcj02',
      success: (res) => resolve({ lat: res.latitude, lon: res.longitude }),
      fail: () => resolve(null)  // 失败静默处理
    });
  });
}
```

## 7. 安全规范

### 7.1 内容安全

1. 图片上传后调用微信云 `imgSecCheck` 接口检测，不通过则拒绝存储
2. 文字内容调用 `msgSecCheck` 接口检测，不通过则拒绝发布
3. 审核服务异常时默认通过（避免误伤），异常日志写入监控系统
4. 管理员后台可手动下架内容、封禁用户

### 7.2 接口安全

1. 所有写操作（POST/PATCH/DELETE）必须携带有效 JWT Token
2. 管理员接口额外校验 `role === 'admin'`
3. 图片上传限制：单文件 ≤ 10MB，格式限 jpg/png/webp，每用户每小时最多20张
4. 所有接口启用 `express-rate-limit`，防止滥用

### 7.3 数据安全

1. 用户 openid 不对外暴露，API 响应中仅返回 `user_id`
2. JWT 有效期 7 天，敏感操作需重新鉴权
3. 所有密钥通过环境变量注入，`.env` 文件列入 `.gitignore`，禁止提交至代码仓库

## 8. 部署方案

### 8.1 推荐方案：微信云开发

1. **云函数**：Node.js 业务逻辑部署为云函数，自动扩缩容
2. **云数据库**：托管 MySQL，无需自行运维
3. **云存储**：替代 COS，用于图片存储
4. **优点**：与微信生态深度集成，免运维，适合校园级小程序快速上线

### 8.2 备选方案：自建服务器

| 步骤       | 操作                                      |
| :------- | :-------------------------------------- |
| 1. 服务器   | 购买云服务器（2核4G），部署 Ubuntu 22.04            |
| 2. 环境    | 安装 Node.js 18、MySQL 8.0、Redis 7.0、Nginx |
| 3. 进程管理  | `pm2 start app.js --name flower-api`    |
| 4. 反向代理  | Nginx 配置 HTTPS，将 `/v1` 代理到 Node 服务      |
| 5. SSL证书 | Let's Encrypt 免费证书（certbot）             |
| 6. 域名配置  | 微信公众平台配置服务器域名白名单                        |

### 8.3 小程序发布流程

1. 微信开发者工具 → 上传代码（填写版本号和描述）
2. 微信公众平台 → 版本管理 → 提交审核
3. 审核通过（通常1-3个工作日）→ 发布上线
4. 通知用户搜索小程序名称或扫码体验
